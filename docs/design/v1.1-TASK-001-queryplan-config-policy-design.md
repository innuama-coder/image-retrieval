## Scope And Boundary

### In Scope

- Canonical v1.1 QueryPlan input and normalized QueryPlan contract.
- Runtime config contract for providers, retrieval channels, Qwen 3.5 VLM, policy,
  quality, output, and redaction.
- Default value rules for `required_image_count`, `quality`,
  `candidate_target`, `retrieval_batch_target`, `retry_limit`,
  `full_attempt_limit`, `retry_count`, and `full_attempt_count`.
- Admission validation, machine-readable diagnostics, error codes, and
  redaction rules.
- Policy config for credentials, paid channel enablement, robots/site-rule,
  login/paywall behavior, authorization risk, and production fixture blocking.
- Handoff contracts consumed by TASK-002 through TASK-005.

### Non-Goals And Forbidden Scope

- Forbidden: implementing Rust code, tests, Cargo manifest changes, provider
  adapters, retrieval channels, Qwen 3.5 VLM calls, orchestrator logic, package
  writing, or CLI behavior.
- This design does not choose the default broad search provider.
- This design does not implement the production Qwen 3.5 VLM adapter; it defines
  the config contract the adapter consumes.
- This design does not authorize paid retrieval; it defines disabled-by-default
  policy and blocker behavior.
- This design does not resolve final robots/site-rule policy; it defines config
  and diagnostics so unresolved posture is explicit.

## Current Source Delta

The current `src/domain/query_plan.rs` already has useful foundations:

- `QueryPlanInput.description`
- `required_count` with default `1`
- `quality_tier` with default `QualityTier::General`
- `retry_limit` with default `3`
- `TaskPlan::from_validated` derives `candidate_target = required_count * 20`,
  `retrieval_batch_target = required_count * 2`, and `max_attempts = 1 + retry_limit`
- input diagnostics that avoid echoing credential-like text

The v1.1 gap is that these fields are still an MVP planning shape. The
production design needs a canonical `required_image_count` field, richer
quality requirements, query expansion fields, provider/retrieval policies,
runtime config, typed policy diagnostics, and explicit attempt-counter
semantics that downstream modules can serialize into the package.

## QueryPlan Interface And Type Contract

### Raw Input DTO

`QueryPlanInput` remains the user-facing serde DTO. v1.1 canonical JSON uses
`required_image_count`; current `required_count` is accepted as an alias during
transition and must not appear as the canonical serialized output field.

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct QueryPlanInput {
    #[serde(default)]
    pub description: String,

    #[serde(default)]
    pub query_texts: Vec<String>,

    #[serde(default = "default_required_image_count", alias = "required_count")]
    pub required_image_count: u32,

    #[serde(default)]
    pub quality: QualityTier,

    #[serde(default)]
    pub quality_requirements: QualityRequirements,

    #[serde(default)]
    pub material_types: Vec<String>,

    #[serde(default)]
    pub visual_requirements: Vec<String>,

    #[serde(default)]
    pub negative_scope: Vec<String>,

    #[serde(default)]
    pub source_diversity_requirement: Option<u32>,

    #[serde(default)]
    pub provider_policy: QueryProviderPolicy,

    #[serde(default)]
    pub retrieval_policy: QueryRetrievalPolicy,

    #[serde(default = "default_retry_limit")]
    pub retry_limit: u8,
}
```

Default and validation rules:

- `description` is required after trimming. Empty or whitespace-only input is
  `INPUT_DESCRIPTION_MISSING`.
- `query_texts` defaults to `[description]` after admission when absent. Empty
  strings are removed with warning `QUERY_TEXT_EMPTY_IGNORED`.
- `required_image_count` defaults to `1`. Explicit `0` is not delivered as
  zero; admission applies `1` and emits warning `REQUIRED_COUNT_ZERO_DEFAULTED`.
- `quality` defaults to `general`; supported serialized values are `general`,
  `high`, and `strict`.
- `quality_requirements` defaults from `quality`. Explicit subfields override
  tier defaults.
- `material_types`, `visual_requirements`, and `negative_scope` default to
  empty arrays. They are preserved for search, quality, retrieval, and review.
- `source_diversity_requirement` defaults to `None`; if present it must be
  between `1` and `required_image_count`.
- `retry_limit` defaults to `3`; values above `3` are rejected with
  `RETRY_LIMIT_EXCEEDED`.

### Quality DTO

`QualityTier` should remain for compatibility with existing source. v1.1 adds
`QualityRequirements` so future quality tasks can use structured thresholds.

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct QualityRequirements {
    #[serde(default)]
    pub minimum_width: Option<u32>,
    #[serde(default)]
    pub minimum_height: Option<u32>,
    #[serde(default)]
    pub allow_watermark: Option<bool>,
    #[serde(default)]
    pub allow_thumbnail_only: bool,
    #[serde(default)]
    pub min_visual_relevance_score: Option<f32>,
}
```

Tier-derived defaults:

| Quality tier | minimum size | thumbnail-only | subjective posture |
| --- | --- | --- | --- |
| `general` | no hard minimum unless config provides one | false | reject obvious mismatch/corruption |
| `high` | config default high threshold if configured | false | prefer clear subject and usable composition |
| `strict` | config default strict threshold if configured | false | reject ambiguous and boundary cases |

`allow_thumbnail_only` must default to `false` because metadata-only and
thumbnail-only delivery is forbidden by PRD AC-013 and the completion design.

### Normalized QueryPlan DTO

Downstream modules must consume `NormalizedQueryPlan`, not raw input.

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct NormalizedQueryPlan {
    pub query_plan_id: QueryPlanId,
    pub description: String,
    pub query_texts: Vec<String>,
    pub required_image_count: u32,
    pub quality: QualityTier,
    pub quality_requirements: QualityRequirements,
    pub material_types: Vec<String>,
    pub visual_requirements: Vec<String>,
    pub negative_scope: Vec<String>,
    pub source_diversity_requirement: Option<u32>,
    pub candidate_target: u32,
    pub retrieval_batch_target: u32,
    pub retry_limit: u8,
    pub full_attempt_limit: u8,
    pub provider_policy: QueryProviderPolicy,
    pub retrieval_policy: QueryRetrievalPolicy,
    pub admission_diagnostics: Vec<AdmissionDiagnostic>,
}
```

Derivation rules:

- `candidate_target = required_image_count * 20`.
- `retrieval_batch_target = required_image_count * 2`.
- `retry_limit = 3` by default and never above `3`.
- `full_attempt_limit = 1 + retry_limit`.
- `retry_count` is a runtime counter and must equal
  `full_attempt_count - 1`.
- `full_attempt_count` starts at `1`, not `0`.
- Maximum normal run attempts are `full_attempt_count in 1..=4`.

Use checked or saturating multiplication with a diagnostic:

- If multiplication overflows `u32`, reject with `TARGET_DERIVATION_OVERFLOW`.
- If values exceed configured `limits.max_required_image_count`, reject with
  `REQUIRED_COUNT_LIMIT_EXCEEDED`.

## Runtime Config Interface And DTO Contract

`RuntimeConfig` is external configuration. It must be loadable from TOML and
JSON where feasible, but the Rust DTO must be serde-ready and independent of
CLI parsing.

```rust

    pub vlm_evaluation: VlmEvaluationConfig,
    #[serde(default)]
    pub policy: PolicyConfig,
    #[serde(default)]
    pub quality_defaults: QualityDefaultsConfig,
    #[serde(default)]
    pub output: OutputConfig,
    #[serde(default)]
```

### Provider Config

```rust
pub struct SearchProviderConfig {
    pub provider_id: String,
    pub provider_kind: SearchProviderKind,
    #[serde(default)]
    pub enabled: bool,
    #[serde(default = "default_weight")]
    pub weight: u32,
    pub endpoint: Option<String>,
    pub credential_env: Option<String>,
    #[serde(default)]
    pub default_query_params: BTreeMap<String, String>,
}
```

Rules:

- Credentials are referenced by environment variable name only.
- `credential_env` values are configuration metadata and can be serialized;
  resolved credential values must never be serialized.
- `enabled` defaults to `false` for paid or non-local providers unless the
  provider kind explicitly defines a free local default.
- `weight` must be greater than zero when enabled.
- Provider readiness failures must use machine-readable failure codes consumed
  by TASK-002.

### Retrieval Policy Config

```rust
pub struct RetrievalChannelConfig {
    pub channel_id: String,
    pub channel_kind: RetrievalChannelKind,
    pub tier: RetrievalChannelTier,
    #[serde(default)]
    pub enabled: bool,
    pub endpoint: Option<String>,
    pub credential_env: Option<String>,
    #[serde(default)]
    pub max_batch_size: Option<u32>,
}

pub struct QueryRetrievalPolicy {
    #[serde(default)]
    pub allow_paid: bool,
    #[serde(default = "default_true")]
    pub respect_robots: bool,
    #[serde(default)]
    pub allow_login: bool,
    #[serde(default)]
    pub allow_paywalled: bool,
}
```

Default policy:

- `allow_paid = false`.
- `respect_robots = true`.
- `allow_login = false`.
- `allow_paywalled = false`.

Policy precedence:

1. QueryPlan policy may narrow runtime policy.
2. QueryPlan policy must not broaden runtime policy without explicit config
   allowance.
3. Paid channel execution requires both runtime enablement and query policy
   `allow_paid = true`.
4. Login/paywall bypass is never implicit; it is blocked unless explicitly
   allowed by both config and policy.

### Qwen 3.5 VLM Config

```rust
pub enum VlmEvaluatorKind {
    Qwen35Vlm,
    Fixture,
    Custom(String),
}

pub struct VlmEvaluationConfig {
    pub provider_id: String,
    pub provider_kind: VlmEvaluatorKind,
    pub model: String,
    #[serde(default)]
    pub enabled: bool,
    pub base_url: Option<String>,
    pub endpoint: Option<String>,
    pub credential_env: Option<String>,
    pub candidate_prompt_template: Option<String>,
    pub image_prompt_template: Option<String>,
    #[serde(default)]
    pub timeout_seconds: Option<u64>,
    #[serde(default)]
    pub fixture_mode: bool,
}
```

Rules:

- Production defaults to `provider_id = "qwen_3_5_vlm"`,
  `provider_kind = Qwen35Vlm`, `model = "qwen-3.5"`, and
  `credential_env = "QWEN_API_TOKEN"` unless runtime config overrides the
  metadata.
- Production mode requires `enabled = true`, configured endpoint/base URL,
  credential env var readiness, and prompt templates for candidate and image
  evaluation.
- The token value is resolved only inside the adapter and must never be
  serialized, logged, or embedded in package evidence.
- `fixture_mode = true` is allowed only in tests or explicit fixture runs; it
  must not produce production package evidence.
- Missing production Qwen 3.5 VLM is `VLM_EVALUATION_UNAVAILABLE` and later quality tasks
  must surface it as execution blocked.

## Control Flow

Admission sequence:

1. CLI loads raw QueryPlan JSON.
2. CLI loads RuntimeConfig from config file and environment references.
3. Config loader parses serde DTOs without resolving secret values into public
   structs.
4. Admission validates QueryPlan input:
   - semantic description
   - required count and quality defaults
   - query text normalization
   - retry limit
   - policy broadening checks
   - sensitive pattern scan
5. Admission validates RuntimeConfig:
   - enabled provider/channel records
   - positive weights
   - credential env var presence by name
   - paid channel gating
   - Qwen 3.5 VLM production readiness
   - output directory policy
6. Admission derives `NormalizedQueryPlan`.
7. Admission returns either `AdmissionOutcome::Accepted` or
   `AdmissionOutcome::Rejected`.

```rust
pub enum AdmissionOutcome {
    Accepted {
        query_plan: NormalizedQueryPlan,
        runtime_config: RuntimeConfig,
        warnings: Vec<AdmissionDiagnostic>,
    },
    Rejected {
        diagnostics: Vec<AdmissionDiagnostic>,
    },
}
```

No search provider, retrieval channel, Qwen 3.5 VLM evaluation, or delivery package
write can start before `AdmissionOutcome::Accepted`.

## Data Flow

```text
QueryPlanInput JSON
  -> serde DTO
  -> validate/admit
  -> NormalizedQueryPlan
  -> SearchRequest planning in TASK-002
  -> Quality request metadata in TASK-003
  -> Retrieval batch planning in TASK-004
  -> RunState and package metadata in TASK-005
```

`RuntimeConfig` flows separately:

```text
config file + env var names
  -> RuntimeConfig DTO
  -> ConfigReadinessReport
  -> provider/channel/Qwen 3.5 VLM/policy readiness
  -> self-check and orchestrator gating
```

The normalized QueryPlan and redacted config summary are package-safe. Raw
secret values, resolved tokens, cookies, and API keys are not package-safe and
must not cross into delivery artifacts, logs, or diagnostics.

## State And Persistence

TASK-001 does not introduce durable storage. State is in memory until TASK-005
materializes the delivery package. The following fields must be serializable in
package metadata later:

```rust
pub struct AttemptCounterState {
    pub full_attempt_count: u8,
    pub retry_count: u8,
    pub full_attempt_limit: u8,
    pub retry_limit: u8,
}
```

Invariants:

- Before first attempt: `full_attempt_count = 1`, `retry_count = 0`.
- During attempt N: `retry_count = full_attempt_count - 1`.
- `full_attempt_limit = 1 + retry_limit`.
- Default `retry_limit = 3`, default `full_attempt_limit = 4`.
- Validation fails package metadata where `retry_count != full_attempt_count - 1`.

Persistence contract for downstream package writers:

- Store `NormalizedQueryPlan` or a redacted projection in
  `package-summary.json` and `retrieval-manifest.json`.
- Store admission warnings and config readiness in `diagnostics/` or
  `package-summary.json`.
- Do not store resolved credential values or raw secret-looking input.

## Errors Diagnostics And Failure Codes

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AdmissionDiagnostic {
    pub code: AdmissionFailureCode,
    pub severity: DiagnosticSeverity,
    pub field_path: String,
    pub message: String,
    pub remediation: Option<String>,
    pub default_applied: Option<String>,
    pub redacted: bool,
}
```

Required codes:

| Code | Severity | Meaning |
| --- | --- | --- |
| `INPUT_DESCRIPTION_MISSING` | error | Description is absent or whitespace-only. |
| `REQUIRED_COUNT_ZERO_DEFAULTED` | warning | Explicit zero was replaced with default one. |
| `REQUIRED_COUNT_LIMIT_EXCEEDED` | error | Required image count exceeds configured local limit. |
| `QUALITY_UNSUPPORTED` | error | Quality value is outside `general/high/strict`. |
| `RETRY_LIMIT_EXCEEDED` | error | Retry limit exceeds 3. |
| `TARGET_DERIVATION_OVERFLOW` | error | Derived candidate or retrieval target overflows. |
| `CONFIG_PROVIDER_WEIGHT_INVALID` | error | Enabled provider has zero weight. |
| `CONFIG_CREDENTIAL_ENV_MISSING` | warning or error | Required environment variable is missing by name. |
| `POLICY_PAID_UNCONFIRMED` | blocker | Paid channel is required but not explicitly allowed. |
| `POLICY_ROBOTS_UNDECIDED` | warning or blocker | Robots/site-rule behavior is unknown under current config. |
| `SENSITIVE_INPUT_REDACTED` | warning | Credential-like input was detected and redacted. |
| `VLM_EVALUATION_UNAVAILABLE` | blocker | Production subjective evaluation cannot run. |

Diagnostics must be machine-readable and human-readable. They may include field
paths and safe enum values, but must not include raw credential, token, cookie,
or private key content.

## Security Authorization And Redaction

Security rules:

- Credentials are externalized through env var names.
- Runtime config stores `credential_env`, not credential values.
- Resolved credentials live only inside provider/channel/Qwen 3.5 VLM adapters and
  are never added to domain DTOs.
- Sensitive redaction applies to QueryPlan text, config string fields, provider
  evidence, channel evidence, diagnostics, package files, audit logs, and trace
  fields.
- Unknown authorization is allowed only with retained risk evidence.
- Explicit prohibited sources are locally rejected.
- Access-restricted sources must not be bypassed by fallback.
- Paid retrieval is blocked unless both runtime config and QueryPlan policy
  allow it.

Redaction must use a shared helper from `src/policy/mod.rs` or an equivalent
central function. A development implementation may improve matching, but it
must preserve the contract that sensitive raw values are not echoed.
